2. 기타 주석 수정

---------------------------------------------------------------------------------------------------------------------------------
2.01 릴리즈 노트 - 김주태

1. 에셋 CheckPoint 추가
	작업중 오류가 발생했을때 발생 위치를 저장하여 다음 실행시 그 위치부터 실행하도록 함.
	에셋에 추가 (예: RPA_AA001_체크포인트)
	설정파일 CheckPoint Value에 에셋명 추가 
	설정파일에 없으면 사용하지 않음.
	정상종료되면 -값으로 초기화 됨.
	주의사항: 	저장되는 값은 다음 실행에도 고유식별 가능한 값이어야 함.
				순서를 고려하여 저장 된 값 이후는 처리대상에 포함되도록 해야함.	
2. 에셋 업무 담당자 추가
	작업 결과 전송시 업무 담당자 사번을 에셋에서 가져올수 있음
	에셋에 추가 (예: RPA_AA001_ADM_업무담당자사번)
	설정파일 AdminIds Asset명에 에셋명 추가 
	테스트를 위해 파일에 직접 입력해서 사용하려면 Value에 입력하고 에셋명을 공백으로 함.
	주의사항: 	복수의 담당자가 있을 경우 콤마로 구분 함.
	
3. 에셋 RPA 시스템 관리자 추가
	시스템 오류 결과 전송시 RPA 시스템 운영자 사번을 에셋에서 가져올수 있음
	에셋에 추가 (예: 공용_ADM_RPA운영자사번)
	설정파일 RPAAdminIds Asset명에 에셋명 추가 
	테스트를 위해 파일에 직접 입력해서 사용하려면 Value에 입력하고 에셋명을 공백으로 함.
	주의사항: 	복수의 담당자가 있을 경우 콤마로 구분 함.

4. 에셋 기본 성공 메시지 추가
	성공 결과 메시지 전송시 전송될 기본 메시지를 에셋에서 가져올수 있음
	에셋에 추가 (예: 공용_MSG_SUCCESS)
	설정파일 MessageSuccess Asset명에 에셋명 추가 
	테스트를 위해 파일에 직접 입력해서 사용하려면 Value에 입력하고 에셋명을 공백으로 함.
	주의사항: 	개행이 필요할 경우 \n을 입력함
	
5. 에셋 기본 오류 메시지 추가
	성공 오류 메시지 전송시 전송될 기본 메시지를 에셋에서 가져올수 있음
	에셋에 추가 (예: 공용_MSG_FAIL)
	설정파일 MessageFail Asset명에 에셋명 추가 
	테스트를 위해 파일에 직접 입력해서 사용하려면 Value에 입력하고 에셋명을 공백으로 함.
	주의사항: 	개행이 필요할 경우 \n을 입력함
	
6. 설정파일에 에셋에서 가져올수 있게 설정 가능함
	설정파일에 개발자가 자유롭게 설정값을 추가할수 있고, 파일로 관리 또는 에셋으로 관리를 선택할수 있음
	설정파일 Asset명에 에셋명을 입력하면 그 에셋명으로 에셋에서 값을 가져와 Value에 추가함. 
	파일에 직접 입력해서 사용하려면 Value에 입력하고 에셋명을 공백으로 함.
	
7. 예외시 처리 방법은 일괄처리함. 예외목록에서 개별 예외별로 처리하지 않음.
	
8. 건수 표시
	오류 발생시: "처리대상 건수: 100 정상 처리 건수: 90 시스템 오류 건수: 5 비즈니스 오류 건수: 4"  
	정상 처리시: "처리대상 건수: 100"
